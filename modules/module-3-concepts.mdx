## Three things a Space gives you

Every Hugging Face Space running an OpenEnv environment provides three access methods automatically — no extra configuration required.

| Component | What it provides | How to access |
|-----------|------------------|---------------|
| **Server** | Running environment endpoint | `https://<username>-<space-name>.hf.space` |
| **Repository** | Pip-installable Python package | `pip install git+https://huggingface.co/spaces/<space>` |
| **Registry** | Docker container image | `docker pull registry.hf.space/<space>:latest` |

One deployment. Three ways to use it.

## Local development with Uvicorn

The fastest iteration loop: clone a Space and run it locally.

```bash
# Clone from HF Space
git clone https://huggingface.co/spaces/openenv/echo-env
cd echo-env

# Install and run
uv sync
uv run server
```

Or with Uvicorn directly:

```bash
uvicorn echo_env.server.app:app --host 0.0.0.0 --port 8000 --reload
```

<Tip>
  The `--reload` flag restarts the server when you change code. Essential for development.
</Tip>

Verify the server is running:

```bash
curl http://localhost:8000/health
# {"status": "healthy"}
```

Connect from Python:

```python
with EchoEnv(base_url="http://localhost:8000").sync() as env:
    result = env.reset()
```

## Docker deployment

Docker gives you isolation and reproducibility. Choose the approach that fits your workflow.

<Tabs>
  <Tab title="Pull from registry">
    Pull the pre-built image directly from a Space's container registry:

    ```bash
    docker pull registry.hf.space/openenv-echo-env:latest
    docker run -d -p 8000:8000 registry.hf.space/openenv-echo-env:latest
    ```
  </Tab>
  <Tab title="Build from source">
    Clone the Space and build the image locally:

    ```bash
    git clone https://huggingface.co/spaces/openenv/echo-env
    cd echo-env
    docker build -t my-echo-env:latest -f server/Dockerfile .
    docker run -d -p 8000:8000 my-echo-env:latest
    ```
  </Tab>
  <Tab title="With environment variables">
    Pass configuration at runtime using `-e` flags:

    ```bash
    docker run -d -p 8000:8000 \\
        -e WORKERS=4 \\
        -e MAX_CONCURRENT_ENVS=100 \\
        my-echo-env:latest
    ```
  </Tab>
</Tabs>

## Deploying to HF Spaces

### `openenv push`

The fastest path from local code to a running endpoint:

```bash
cd my_env
openenv push --repo-id username/my-env
```

Your environment is now live at four URLs:

| URL | What it serves |
|-----|----------------|
| `https://username-my-env.hf.space` | API endpoint |
| `https://username-my-env.hf.space/web` | Web UI |
| `https://username-my-env.hf.space/docs` | API docs |
| `https://username-my-env.hf.space/health` | Health check |

### The `openenv.yaml` manifest

The `openenv.yaml` file at the root of your environment controls Space settings:

```yaml
name: my_env
version: "1.0.0"
description: My custom environment
```

### Environment variables

Configure your deployment via **Space Settings → Variables**:

| Variable | Default | Description |
|----------|---------|-------------|
| `WORKERS` | 4 | Uvicorn worker processes |
| `PORT` | 8000 | Server port |
| `HOST` | 0.0.0.0 | Bind address |
| `MAX_CONCURRENT_ENVS` | 100 | Max WebSocket sessions |

### Hardware options

| Tier | vCPU | RAM | Cost |
|------|------|-----|------|
| CPU Basic (Free) | 2 | 16GB | Free |
| CPU Upgrade | 8 | 32GB | $0.03/hr |

<Note>
  The free tier handles ~128 concurrent sessions — enough for development and demos.
</Note>

## The full workflow

<Steps>
  <Step title="Scaffold your environment">
    Create a new environment from the template:

    ```bash
    openenv init my_env
    ```
  </Step>
  <Step title="Implement your logic">
    Edit `server/environment.py` to define `reset`, `step`, and `close`.
  </Step>
  <Step title="Test locally">
    Run the development server and verify your environment behaves correctly:

    ```bash
    uv run server
    ```
  </Step>
  <Step title="Deploy to HF Spaces">
    Push your environment to a Hugging Face Space with one command:

    ```bash
    openenv push
    ```
  </Step>
  <Step title="Install the client">
    Install the client package directly from your Space's repository:

    ```bash
    pip install git+https://huggingface.co/spaces/username/my-env
    ```
  </Step>
</Steps>

## Choosing your access method

| Method | Use when | Pros | Cons |
|--------|----------|------|------|
| **Remote Space** | Quick testing, low volume | Zero setup | Network latency |
| **Local Docker** | Development, high throughput | Full control, no network | Requires Docker |
| **Local Uvicorn** | Fast iteration | Fastest reload | No isolation |
