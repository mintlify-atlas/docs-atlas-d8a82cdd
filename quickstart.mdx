## Install

<Steps>
  <Step title="Install openenv-core">
    ```bash
    pip install openenv-core
    ```
  </Step>
  <Step title="Clone the OpenEnv repo for typed clients">
    The typed environment clients live in the OpenEnv repository:
    ```bash
    git clone https://github.com/meta-pytorch/OpenEnv.git
    ```
    Then add both paths to your Python path:
    ```python
    import sys, os
    repo = os.path.abspath('OpenEnv')
    sys.path.insert(0, repo)
    sys.path.insert(0, os.path.join(repo, 'src'))
    ```
  </Step>
</Steps>

## Connect to your first environment

OpenEnv has two environment types. Choose one to start:

<Tabs>
  <Tab title="MCP Tool-Based (Echo)">
    The Echo environment uses the MCP tool-calling interface — good for understanding tool-based interaction:

    ```python
    from envs.echo_env import EchoEnv

    with EchoEnv(base_url="https://openenv-echo-env.hf.space").sync() as env:
        env.reset()
        response = env.call_tool("echo_message", message="Hello, OpenEnv!")
        print(response)  # Hello, OpenEnv!
    ```

    MCP environments expose `list_tools()` and `call_tool(name, **kwargs)` instead of `step()`.
  </Tab>
  <Tab title="Standard RL (OpenSpiel Catch)">
    The standard interface uses `reset()`, `step()`, and `state()`. This is what most environments use:

    ```python
    from envs.openspiel_env import OpenSpielEnv
    from envs.openspiel_env.models import OpenSpielAction

    with OpenSpielEnv(base_url="https://openenv-openspiel-catch.hf.space").sync() as env:
        result = env.reset()
        print(result.observation.legal_actions)  # e.g. [0, 1, 2]

        result = env.step(OpenSpielAction(action_id=1, game_name="catch"))
        print(result.observation.info_state)     # Game state vector
        print(result.done)                        # False (game continues)
    ```
  </Tab>
</Tabs>

## Run a full episode

Play a complete game of Catch with a simple heuristic policy:

```python
import random
from envs.openspiel_env import OpenSpielEnv
from envs.openspiel_env.models import OpenSpielAction

def random_policy(obs):
    return random.choice(obs.legal_actions)

with OpenSpielEnv(base_url="https://openenv-openspiel-catch.hf.space").sync() as env:
    result = env.reset()
    steps = 0

    while not result.done:
        action_id = random_policy(result.observation)
        result = env.step(OpenSpielAction(action_id=action_id, game_name="catch"))
        steps += 1

    print(f"Game over after {steps} steps. Reward: {result.reward}")
```
